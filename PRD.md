# PRD — 인천공항 혼잡도 대시보드 (V2)

## 0. 문서 정보
- 버전: v2 (프론트엔드 직접 API 호출 방식)
- 관련 이전 버전: `../incheon` (Supabase Edge Function 프록시 방식) — 아키텍처 비교는 [부록 A](#부록-a-이전-incheon-버전과의-차이) 참고
- 작성일: 2026-07-23
- 배포 대상: Vercel

## 1. 목적
공공데이터포털이 제공하는 인천공항 여객 혼잡도 데이터를 조회하여, 사용자가 원하는 날짜·시간의 터미널(T1/T2)별 입국장·출국장 혼잡도를 숫자와 그래프로 확인할 수 있는 웹 대시보드를 만든다. 로그인한 사용자는 자주 조회하는 시간대 조합을 즐겨찾기로 저장해 반복 조회를 빠르게 할 수 있다.

## 2. 대상 사용자
- 인천공항 이용 예정자 (출국/입국 시간대 혼잡도를 미리 확인하고 싶은 일반 사용자)
- 반복적으로 같은 시간대(예: 매번 이용하는 항공사 출발 시간대)를 확인하는 재방문 사용자

## 3. 용어 정의
| 용어 | 정의 |
|---|---|
| 터미널(Terminal) | `T1`, `T2` 중 하나 |
| 게이트 타입(GateType) | `입국장(arrival)`, `출국장(departure)` |
| 혼잡도(CongestionLevel) | 0~100 사이 수치 또는 원천 API가 제공하는 혼잡도 지표 |
| 혼잡도 라벨 | `원활` / `보통` / `혼잡` 등 API 또는 자체 기준으로 분류한 등급 |
| 즐겨찾기(FavoriteSlot) | 로그인 사용자가 저장한 "날짜+시간+터미널+게이트타입" 조합 |

## 4. 핵심 기능 요구사항

### 4.1 오늘/내일 데이터 불러오기
- 앱 진입 시 공공데이터 API에서 **오늘** 날짜의 혼잡도 데이터를 기본으로 불러온다.
- 사용자는 "오늘/내일" 탭(또는 토글)으로 전환하여 내일 데이터를 조회할 수 있다.
- API 응답은 프론트엔드에서 짧게 메모리 캐시하여, 같은 날짜를 반복 요청할 때 불필요한 재호출을 줄인다 (탭 전환 시 재호출 방지 정도의 단순 캐시).
- 데이터 로딩 중 스켈레톤/로딩 인디케이터, 실패 시 에러 메시지 + 재시도 버튼을 표시한다.

### 4.2 날짜·시간 선택 후 상세 보기
- 날짜 선택기(오늘/내일 범위 내, API가 지원하는 범위로 제한)와 시간 선택기(시 단위 또는 API가 제공하는 시간 단위)를 제공한다.
- 선택한 날짜·시간에 해당하는 T1/T2 × 입국장/출국장 = 4가지 조합의 혼잡도를 카드 형태 숫자로 보여준다.
- 각 카드에는 수치, 등급 라벨, 등급별 색상(원활=녹색 계열, 보통=노랑 계열, 혼잡=빨강 계열)을 표시한다.

### 4.3 터미널별 비교 그래프
- 선택한 날짜 기준, 시간대별 혼잡도 추이를 T1 vs T2로 비교하는 그래프(라인 또는 바 차트)를 제공한다.
- 입국장/출국장을 별도 그래프 또는 그래프 내 시리즈로 구분해 비교 가능해야 한다.
- 그래프 라이브러리는 React 생태계 표준 차트 라이브러리(예: recharts)를 사용한다.

### 4.4 다크모드 전환
- 헤더에 라이트/다크 모드 토글을 제공한다.
- 선택한 모드는 `localStorage`에 저장하여 재방문 시에도 유지된다.
- 시스템 설정(`prefers-color-scheme`)을 최초 기본값으로 사용하고, 사용자가 수동 전환하면 그 값을 우선한다.

### 4.5 회원가입/로그인 및 즐겨찾기
- Supabase Auth(이메일/비밀번호)로 회원가입, 로그인, 로그아웃을 지원한다.
- 로그인한 사용자는 현재 조회 중인 "날짜+시간+터미널+게이트타입" 조합을 즐겨찾기로 저장할 수 있다.
- 즐겨찾기 목록 화면에서 본인이 저장한 즐겨찾기만 조회하고, 선택 삭제할 수 있다.
- 즐겨찾기 항목 클릭 시 해당 날짜·시간 상세 화면으로 즉시 이동한다.
- 비로그인 사용자는 즐겨찾기 저장 버튼 클릭 시 로그인을 유도하는 안내를 표시한다.

## 5. 화면 구성 (정보 구조)
1. **헤더**: 서비스 타이틀, 다크모드 토글, 로그인/로그아웃 버튼(또는 사용자 이메일 표시)
2. **날짜/시간 선택 영역**: 오늘/내일 탭, 날짜 선택기, 시간 선택기
3. **혼잡도 상세 카드**: T1/T2 × 입국장/출국장 4개 카드 + 즐겨찾기 저장 버튼
4. **터미널 비교 그래프**: 시간대별 T1 vs T2 추이 그래프
5. **즐겨찾기 목록** (로그인 시에만 노출): 저장된 즐겨찾기 리스트, 각 항목에 삭제 버튼

## 6. 데이터 모델

### 6.1 프론트엔드 타입 (참고용, 이전 `incheon` 프로젝트 타입 재사용)
```ts
type Terminal = 'T1' | 'T2';
type GateType = 'arrival' | 'departure';
type CongestionLabel = '원활' | '보통' | '혼잡';

interface CongestionRecord {
  id: string;
  date: string;   // YYYY-MM-DD
  time: string;   // HH:mm
  terminal: Terminal;
  gateType: GateType;
  congestionLevel: number;      // 0-100
  congestionLabel: CongestionLabel;
}

interface FavoriteSlot {
  id: string;
  userId: string;
  date: string;
  time: string;
  terminal: Terminal;
  gateType: GateType;
  createdAt: string;
}

interface AuthUser {
  id: string;
  email: string;
}
```

### 6.2 Supabase 데이터베이스 — 즐겨찾기 테이블
- 테이블명: `favorites` (가칭, 구현 시 확정)
- 컬럼: `id`(uuid, PK), `user_id`(uuid, FK → `auth.users.id`), `date`, `time`, `terminal`, `gate_type`, `created_at`
- **RLS(행 수준 보안) 정책**:
  - RLS를 반드시 활성화한다.
  - `SELECT`, `INSERT`, `DELETE` 정책 모두 `auth.uid() = user_id` 조건으로 제한하여, 사용자는 본인 소유 행만 조회·생성·삭제할 수 있다.
  - `UPDATE`는 이번 범위(조회/삭제)에서는 필요하지 않으므로 기본 비허용으로 둔다.

## 7. 외부 연동 — 공공데이터포털 인천공항 여객 혼잡도 API
- 이전 `incheon` 프로젝트와 동일한 공공데이터포털 API를 사용한다 (정확한 서비스명/엔드포인트/요청 파라미터는 구현 착수 시 공공데이터포털 문서로 재확인).
- **호출 방식**: 이번 버전(V2)은 프론트엔드에서 API를 **직접 호출**한다 (서버리스 프록시를 두지 않음).
- **API 키 관리**:
  - 로컬 개발: `.env`에 `VITE_` 접두사로 키를 두고 `.gitignore`에 포함한다.
  - 배포: Vercel 프로젝트 환경변수로 동일한 키를 등록한다.
  - `.env.example`에 키 이름만 남기고 실제 값은 커밋하지 않는다.

### 7.1 보안 주의사항 (반드시 인지해야 할 트레이드오프)
- Vite로 빌드된 프론트엔드에서 `VITE_` 접두사 환경변수는 **빌드 결과물(JS 번들)에 그대로 포함되어 브라우저에서 누구나 열람 가능**하다. 즉 API 키가 사실상 공개된다.
- 이는 사용자가 확인 후 명시적으로 선택한 방식이며, 완전한 키 은닉이 필요할 경우 추후 서버리스 프록시(Vercel Serverless Function 등)로 전환할 수 있다 (부록 A 참고).
- 완화책(선택 사항, 필수 아님): 공공데이터포털에서 키 발급 시 일일 호출 한도를 낮게 설정, 필요 시 키 재발급/로테이션.

## 8. 인증
- Supabase Auth를 사용하여 이메일/비밀번호 기반 회원가입·로그인·로그아웃을 구현한다.
- 로그인 세션은 Supabase 클라이언트가 기본 제공하는 세션 유지 방식(로컬 스토리지 기반)을 사용한다.
- 프론트엔드 `.env`에는 `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`만 둔다 (Supabase anon key는 RLS 전제하에 공개되어도 되는 키).

## 9. 기술 스택
- React + TypeScript + Vite
- 차트: recharts
- 인증/DB: Supabase (Auth, Database + RLS)
- 배포: Vercel
- 공공데이터포털 API: 프론트엔드 직접 호출 (fetch/axios)

## 10. 환경 변수
| 변수명 | 위치 | 설명 |
|---|---|---|
| `VITE_PUBLIC_DATA_API_KEY` | `.env`, Vercel 환경변수 | 공공데이터포털 발급 키 |
| `VITE_SUPABASE_URL` | `.env`, Vercel 환경변수 | Supabase 프로젝트 URL |
| `VITE_SUPABASE_ANON_KEY` | `.env`, Vercel 환경변수 | Supabase anon(public) key |

- `.env`는 `.gitignore`에 포함하고, `.env.example`로 키 목록만 공유한다.

## 11. 비기능 요구사항
- **반응형**: 모바일/데스크톱 모두에서 카드·그래프 레이아웃이 자연스럽게 보여야 한다.
- **에러 처리**: 공공데이터 API 실패, 인증 실패, 즐겨찾기 저장/삭제 실패 시 사용자에게 명확한 안내 메시지를 표시한다.
- **접근성**: 혼잡도 등급을 색상만으로 구분하지 않고 텍스트 라벨을 함께 표시한다.
- **성능**: 동일 날짜 재조회 시 불필요한 API 재호출을 피한다 (단순 메모리 캐시 수준).

## 12. 개발 순서 원칙
- 화면은 목업 데이터로 먼저 구성한 뒤, 마지막 단계에서 실제 공공데이터 API/Supabase 연동으로 교체한다.
- 데이터 접근 함수는 목업 단계부터 실제 연동 단계까지 동일한 함수 시그니처를 유지한다 (예: `fetchTodayCongestion`, `fetchTomorrowCongestion`, `listFavorites`, `addFavorite`, `removeFavorite`).

## 13. 범위 제외 (Out of Scope)
- 소셜 로그인(OAuth) — 이번 버전은 이메일/비밀번호만 지원
- 비밀번호 재설정/이메일 인증 플로우 상세 설계 — 필요 시 별도 문서에서 다룸
- 반복 요일/시간 패턴 즐겨찾기 — 이번 버전은 "날짜+시간+터미널+게이트타입" 단건 저장만 지원
- 과거(어제 이전) 데이터 조회
- 다국어(i18n) 지원
